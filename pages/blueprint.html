Blueprint

Düğüm (node) tabanlı, sürüklenebilir bağlantı diyagramı. Her bağlantı kartın merkezinden değil, belirli port noktalarından çizilir. Veritabanı şemaları, veri akışları ve mimari diyagramları için uygundur.

Örnek — Veritabanı Şeması

FK alanları solda input, PK alanları sağda output port olarak gösterilir. Bağlantılar companies.id → users.company_id gibi spesifik port noktalarından çizilir.

Kullanım

<c-blueprint> <script type="application/json"> { "nodes": [ { "id": "users", "title": "users", "x": 20, "y": 40, "color": "#1456c8", "ports": [ { "id": "company_id", "label": "company_id", "dir": "in", "type": "UUID", "color": "#e6a800" }, { "id": "id", "label": "id", "dir": "out", "type": "UUID", "color": "#4488dd" }, { "id": "name", "label": "name", "dir": "none", "type": "VARCHAR(100)" } ] } ], "connections": [ { "from": "companies.id", "to": "users.company_id", "color": "#e6a800" } ] } </script> </c-blueprint>

Node Özellikleri

AlanTürAçıklama
idstringBağlantılarda referans için benzersiz kimlik
titlestringKartın başlık satırında gösterilen metin
x / ynumberBaşlangıç konumu (piksel, scene-local)
colorstringBaşlık arka plan rengi (CSS renk değeri)
portsPort[]Aşağıdaki tabloya bakınız

Port Özellikleri

AlanTürAçıklama
idstringBağlantı hedefi için nodeId.portId formatında kullanılır
labelstringPort satırında gösterilen metin
dir"in" | "out" | "none"in: sol nokta, out: sağ nokta, none: yalnızca etiket
typestringEtiket yanında soluk yazıyla gösterilen tür bilgisi (opsiyonel)
colorstringPort noktasının rengi; belirtilmezse node rengi kullanılır

Bağlantı (connection) Özellikleri

AlanTürAçıklama
fromstringKaynak port — "nodeId.portId" (dir: "out" olmalı)
tostringHedef port — "nodeId.portId" (dir: "in" olmalı)
colorstringBezier eğrisinin rengi
Kartları sürükleyebilirsin — bağlantı eğrileri anlık güncellenir. Boş alana tıklayıp sürükleyerek canvas'ı kaydır, fare tekerleği ile zum yap. Sağ alt köşedeki ↺ butonu tüm kartları başlangıç konumuna sıfırlar.
